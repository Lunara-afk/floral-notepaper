### P1-2 — #297: txt 文件 UTF-8 编码检测问题（v1.0.4）

| 字段 | 内容 |
|------|------|
| **上报人** | asgudao |
| **提交时间** | **2026-06-14** |
| **版本** | v1.0.4 |
| **平台** | Windows 11 |
| **评论** | 1 |
| **复现** | 点击 txt 文件→报错"数据流中没有包含UTF-8文本" |
| **URL** | https://github.com/Achilng/floral-notepaper/issues/297 |

**源码查证**：✅ **确认存在，根源可定位**

`src-tauri/src/lib.rs:105-112`：
```rust
fn read_external_file(path: String) -> Result<String, AppError> {
    let valid_path = validate_path(&path)?;
    std::fs::read_to_string(&valid_path).map_err(|e| AppError { ... })
}
```

**根本原因**：`std::fs::read_to_string` 仅支持 UTF-8。中文 Windows 默认编码是 GBK/GB2312，用记事本创建的 txt 文件为 GBK 编码时直接失败。

**修复建议**：
```rust
// 使用 encoding_rs 库进行编码检测和回退
use encoding_rs::Encoding;
fn read_file_with_fallback(path: &Path) -> Result<String, AppError> {
    let bytes = std::fs::read(path)?;
    if let Ok(s) = std::str::from_utf8(&bytes) {
        return Ok(s.to_owned());
    }
    // 尝试 GBK 解码
    let (cow, _, had_errors) = encoding_rs::GBK.decode(&bytes);
    if had_errors {
        // 进一步尝试其他常见编码
    }
    Ok(cow.into_owned())
}
```

**与已完成修复的关联**：⚪ T2（路径校验）不涉及编码处理，此问题是新增发现。
