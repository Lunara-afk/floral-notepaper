## 2. 本地源码中发现的 Bug 隐患

通过在源码中搜索 `unwrap()`、`expect()`、`unwrap_or_default()` 等行为，发现以下潜在风险点：

### 2.1 `unwrap_or_default()` 时钟回退静默失败

| 字段 | 内容 |
|------|------|
| **文件:行号** | `src-tauri/src/lib.rs:89` |
| **代码** | `.duration_since(std::time::UNIX_EPOCH).unwrap_or_default()` |
| **问题** | 系统时钟在 1970-01-01 之前时静默返回 `0.0` |
| **影响** | 极低（现代操作系统时钟极少在此范围之外） |
| **严重度** | 🟡 LOW |

### 2.2 `unwrap_or(false)` 窗口可见性

| 字段 | 内容 |
|------|------|
| **文件:行号** | `src-tauri/src/desktop.rs:1055` |
| **代码** | `window.is_visible().unwrap_or(false)` |
| **问题** | Tauri 窗口不可见性判断失败时视为隐藏 |
| **影响** | 低 — 最多导致隐藏操作跳过 |
| **严重度** | 🟢 INFO |

### 2.3 `build.rs` panics

| 字段 | 内容 |
|------|------|
| **文件:行号** | `src-tauri/build.rs:2-6` |
| **代码** | 3 处 `.expect()`（读取 tauri.conf.json + 解析 JSON + 获取 version） |
| **问题** | 构建时如果 `tauri.conf.json` 损坏则直接 panic |
| **影响** | 符合预期 — 构建脚本本应快速失败 |
| **严重度** | 🟢 INFO |

### 2.4 `lib.rs:472` 应用构建 panic

| 字段 | 内容 |
|------|------|
| **文件:行号** | `src-tauri/src/lib.rs:472` |
| **代码** | `.expect("error while building tauri application")` |
| **问题** | Tauri 应用构建失败直接 panic |
| **影响** | 符合预期 — 应用无法构建时无意义的兜底 |
| **严重度** | 🟢 INFO |

### 2.5 Scheduler 线程非优雅退出（已修复）

| 字段 | 内容 |
|------|------|
| **文件** | `src-tauri/src/updater/scheduler.rs:14` |
| **问题** | 原代码 `loop {}` 无限循环，应用退出时被强制终止 |
| **严重度** | 🟡 LOW |
| **修复** | ✅ **T9** — 已添加 `SCHEDULER_SHUTDOWN` 标志 + `RunEvent::Exit` 触发 |

### 2.6 7 个命令路径未校验（已修复）

| 字段 | 内容 |
|------|------|
| **文件** | `src-tauri/src/lib.rs:50-223` |

| 本次修复 | 直接关联的 Bug Issue | 间接覆盖的 Bug 场景 |
